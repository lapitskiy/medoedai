# Правила большого эджа для XGB (коротко)

## Цель
- Делать редкие, но сильные входы (`entry_long`), а не частые слабые сигналы.

## Базовый профиль grid
- `max_hold_steps`: `120..240`, шаг `60`
- `min_profit`: `1.5..3.0%`, шаг `0.5%`
- `p_enter_threshold`: `0.70..0.90`, шаг `0.05`
- `fee`: `0.06%` (или реальный round-trip)
- `entry_tp_pct`: `1.8..3.0%`
- `entry_sl_pct`: `-1.2..-0.8%`
- `entry_trail_pct`: фикс или шаг `0.05` (иначе сетка может схлопнуться в 1 значение)
- `scale_pos_weight`: `-1` (auto)

## Правила отбора run
- Брать run только если `f1(1) >= 0.12`
- `pred_non_hold` держать в диапазоне `0.002..0.02`
- `proxy_pnl_val.trades >= 5`
- `proxy_pnl_val.pnl_sum > 0`

## Антипаттерны
- Не ставить слишком высокий `min_profit` при коротком `max_hold` — модель уходит в вечный `hold`.
- Не оценивать только по `val_acc` — при дисбалансе он может быть высоким при нулевом `f1(1)`.
- Не запускать wide-grid без фильтра по `pred_non_hold` и `proxy_pnl`.

## Мини-чек после обучения
- Сначала фильтр по правилам отбора.
- Потом OOS на `30/60/90` днях.
- В прод только модели, где OOS подтверждает стабильность, а не единичный всплеск.
